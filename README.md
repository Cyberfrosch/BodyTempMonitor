# BodyTempMonitor

Мониторинг температуры тела с использованием двух датчиков DS18B20, вывода на LCD 20x4 (I2C) и светодиодной индикации. Устройство предназначено для медицинского контроля (гипотермия, норма, лихорадка, критическая гипертермия) с визуальным оповещением.

![Схема подключения](wiring-diagram.png)

## Особенности

- Измерение температуры двумя независимыми датчиками DS18B20.
- Отображение текущих значений и максимальной температуры на LCD 20x4.
- Определение режима:
  - **Гипотермия** (<35°C)
  - **Норма** (35–37.5°C)
  - **Лихорадка** (37.5–40°C)
  - **Критическая гипертермия** (>40°C)
  - **Ошибка датчиков** (оба не отвечают)
- Светодиодная индикация:
  - Красный светодиод (HEATER_LED) — включение "нагрева" при гипотермии.
  - Синий светодиод (FAN_LED) — включение "вентилятора" при лихорадке.
  - Мигание обоими светодиодами в аварийных режимах (критическом или ошибке).
- Периодический опрос датчиков (2 секунды).
- Вывод отладочной информации в Serial.

## Аппаратное обеспечение

- Микроконтроллер: ESP32, Arduino Uno и т.п. (поддержка библиотек OneWire, DallasTemperature, LiquidCrystal_I2C).
- Датчики: 2 × DS18B20 (подключены по шине OneWire).
- Дисплей: LCD 20x4 с I2C-адаптером (адрес 0x27).
- Светодиоды: 2 шт. (HEATER, FAN) с токоограничивающими резисторами.
- Провода, резистор 4.7 кОм для подтяжки шины OneWire.

## Схема подключения

| Компонент       | Пин ESP32 | Примечание                 |
|-----------------|-----------|----------------------------|
| DS18B20 (DATA)  | 4         | Подтяжка к 3.3В резистором 4.7кОм |
| LCD I2C (SDA)   | 21        |                            |
| LCD I2C (SCL)   | 22        |                            |
| Светодиод HEATER| 19        | Анод через резистор ~220 Ом |
| Светодиод FAN   | 18        | Анод через резистор ~220 Ом |

*Примечание:* Если используется другая плата, уточните пины I2C и доступные выводы.

## Установка и настройка

### Вариант 1: Запуск в симуляторе Wokwi

Проект полностью готов для симуляции на [Wokwi.com](https://wokwi.com) — это позволяет протестировать логику работы без физического оборудования.

> **Онлайн-симуляция**: Готовый проект доступен на Wokwi — [нажмите здесь](https://wokwi.com/projects/457950405254863873) для запуска в браузере без установки оборудования.

1. Перейдите на [Wokwi.com](https://wokwi.com) и нажмите **"New Project"** (Arduino / ESP32).
2. Удалите содержимое по умолчанию и загрузите файлы из этого репозитория:
   - `sketch.ino` — основной файл проекта.
   - `temperature_monitor.h` — заголовочный файл.
   - `temperature_monitor.cpp` — реализация модуля.
   - `diagram.json` — схема подключения (из папки `wokwi` или корня репозитория).
3. ИЛИ вручную создайте файлы и скопируйте код из репозитория, затем вставьте содержимое `diagram.json` в редактор схем.
4. Нажмите **"Start Simulation"** — через несколько секунд вы увидите работу проекта в симуляторе.

> Все необходимые библиотеки (`OneWire`, `DallasTemperature`, `LiquidCrystal I2C`) уже предустановлены в Wokwi, дополнительная установка не требуется.

### Вариант 2: Сборка для реального устройства

#### Требования

- Установленная среда **Arduino IDE** или **PlatformIO**.
- Библиотеки (устанавливаются через менеджер библиотек):
  - **OneWire** (Paul Stoffregen)
  - **DallasTemperature** (Miles Burton)
  - **LiquidCrystal I2C** (Frank de Brabander) или аналог

#### Сборка и загрузка

1. Скачайте или клонируйте репозиторий:
   ```bash
   git clone https://github.com/Cyberfrosch/BodyTempMonitor.git
   ```
2. Откройте файл `sketch.ino` в Arduino IDE.
3. Убедитесь, что все перечисленные библиотеки установлены (меню Скетч → Подключить библиотеку → Управлять библиотеками...).
4. Подключите ESP32 (или другую плату) к компьютеру, выберите тип платы и порт в меню Инструменты.
5. Нажмите кнопку Загрузить (→).

### Структура проекта

Проект разделён на три файла для удобства поддержки:

- `sketch.ino` — главный файл, содержащий `setup()` и `loop()`, включает заголовочный файл модуля.
- `temperature_monitor.h` — заголовочный файл с объявлениями констант, структур, перечислений и прототипов функций.
- `temperature_monitor.cpp` — реализация функций модуля (чтение датчиков, определение режима, управление LCD и светодиодами).

### Конфигурация

Все настраиваемые параметры вынесены в заголовочный файл `temperature_monitor.h`:

- Пины подключения (`TEMP_SENSOR_PIN`, `FAN_LED`, `HEATER_LED`).
- Адрес и размер LCD (`LCD_ADDR`, `LCD_COLS`, `LCD_ROWS`).
- Температурные пороги (`HYPOTHERMIA_THRESHOLD`, `NORMAL_THRESHOLD`, `FEVER_THRESHOLD`).
- Интервалы времени (`MEASURE_INTERVAL_MS`, `BLINK_INTERVAL_MS`).

Измените их при необходимости перед компиляцией.

## Использование

После загрузки прошивки:

1. Дисплей покажет приветствие, затем начнёт отображать температуры.
2. На первой строке — значение первого датчика (`T1`).
3. На второй — второго (`T2`).
4. На третьей — статус (`HYPOTHERMIA`, `NORMAL`, `FEVER`, `CRITICAL!`, `SENSOR ERR`).
5. На четвёртой — максимальная температура из двух датчиков (или сообщение об ошибке).
6. Светодиоды загораются/мигают согласно режиму.

Последовательный монитор (скорость 115200) выводит отладочную информацию.
